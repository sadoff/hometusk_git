Приложение для хранения и записи новых рецептов в lounge-баре.
1) Система представляет собой приложение в котором хранится весь список рецептов необходимых для работы персонала в баре.
Выбрана трёхуровневая архитектура для системы по причине необходимого контроля как за сохранностью уже имеющихся рецептов в бд, так и за добавлением новых. В случае увеличения нагрузки легко масштабируется до многоуровневой архитектуры.
2) Архитектура состоит из следующих компонентов:
- Клиент(фронтенд морда - просмотр рецептов/добавление новых) 
- Сервер приложения (отвечает за бизнес логику - вызов редактора/модерация/уровни доступа для пользователей + общение с БД - отправка запросов/составление запросов/подключение к БД и etc)
- Сервер БД (MySQL)

3) Первое звено (клиент) - html + JS
   Второе звено (бизнес логика) - laravel
   Третье звено (сервер БД) - MySQL
4) Производительность - выбранная архитектура позволяет использовать высокоскоростные сетевые соединения между серверами, и
отдавать на клиент только результат запроса. 
   Безопасность - клиент не имеет прямого доступа к серверу БД(разные доступы пользователей),
   Надёжность - в случае необходимости выбранная архитектура легко масштабируется.
